
About the system built so far:

AIHubBackend is the API layer. (Rest Layer)
FoundryPortal is the frontend.
And yet FoundryPortal UI layer should not directly call the backend.
It should go through a composite layer inside the FoundryPortal /src/api layer.

The Backend APIs are working fine. I tested it.
The frontend is giving problem.

Desired Application flow: 

-User see a default page and can navigate around but there is not much (Home.jsx).
-User must login to get the details.
-User has two choices to login. Microsoft or Google.
-Once user  Sign In, the User stays on the home page.
-There is a top navigation added Home, Inventory, Model Hub, Agent Hub.
-Home Tab is basic information:
-Inventory Tab has Subscription and associated information.
    Things like: Foundry => Project => Model
- The inventory hub loads all the Subscriptions in a dropdown
-Upon selection of a Sub, it should find all the foundries. 
-When user selects a foundry, it should population, location, Resource group, Resource Group Region, and projects.
-Once this is loaded it should load all the Models into Model Information pannel under pooled model.
-The models should showup and are clickable.
-When user clicks the model link it should open in a Modal box.
